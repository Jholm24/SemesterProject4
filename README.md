# ST4-PRO – Cyber-Physical Systems & Integration Project

## Overview
This project is part of the 4th semester Software Technology Engineering program at the **University of Southern Denmark**.  
The objective is to develop a system capable of interfacing with and orchestrating a production line in the SDU I4.0 lab (simulated via Docker), which produces parts for drones.

**Case study:** Interoperability challenges in a production line consisting of a warehouse (**SOAP**), an AGV (**REST**), and an assembly station (**MQTT**).

---

## Learning Objectives
Integrates knowledge from the following modules:
- **ST4-SCFS**: Software Technology in Cyber-Physical Systems (interfacing and protocols).
- **ST4-CPS**: Design of Reliable Cyber-Physical Systems (reliability and safety).
- **SI4-KBS**: Component-Based Systems (architecture and system integration).

---

## Project Case
- **Interfacing:** Handling three distinct communication protocols: **SOAP** (Warehouse), **REST** (AGV), and **MQTT** (Assembly).
- **Orchestration:** Developing components to manage the production sequence from raw parts to the finished stored product.
- **Simulation:** Utilizing a **Docker** environment for development and testing independent of physical hardware.
- **Quality:** Requirements specification based on the **FURPS+** model and a focus on component-based architecture.

---

## Deliverables
- **Requirements Specification:** Functional and non-functional requirements (FURPS+).
- **System Architecture:** Design of integration and production orchestration components.
- **Implementation:** - Components for integrating the three external systems.
    - A simple GUI for monitoring parameters and connection status.
    - A production coordinator to organize and execute sequences.
- **Documentation:** Methodical analysis of choices, consequences, and architectural decisions.

---

## Evaluation
- Based on the ability to solve real-world interoperability challenges.
- Assessment of the system's architecture and integration robustness.
- Oral exam based on the report and the software implementation.
- Grading: **7-point Danish scale**.
